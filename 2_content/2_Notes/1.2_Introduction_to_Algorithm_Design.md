## 1.2.2 What is an Algorithm?

> [!info] Definition
> An **algorithm** is a solution to a **general, well-specified problem**.
>
> Its purpose is to transform **any valid input** into the required output by following a defined sequence of steps.

### How Do We Evaluate an Algorithm?

Algorithms are typically judged using three key characteristics.

| Property    | Description                                           | Example                 |
| ----------- | ----------------------------------------------------- | ----------------------- |
| Correctness | Produces the correct output for every valid input.    | Flight control software |
| Efficiency  | Uses time and memory effectively.                     | Search engines          |
| Generality  | Handles a wide variety of valid inputs and scenarios. | Navigation systems      |

## 1.2.3 Algorithm Efficiency

### Efficiency

> [!info]
> Efficiency measures how effectively an algorithm uses computational resources, primarily:
>
> - Execution time
> - Memory usage

> [!warning]
> Poor efficiency can have serious real-world consequences where software must respond quickly.

### Scalability

> [!info]
> Scalability describes how well an algorithm continues to perform as the amount of data increases.

A suitable algorithm for small datasets may become impractical for larger datasets.

> [!note]
> If performance degrades significantly as input size grows, the algorithm may need to be improved or replaced.

## 1.2.4 Algorithm Properties

Before designing an algorithm, the problem must be clearly specified.

### Problem Specification

| Component | Description |
| --------- | ----------- |
| **Input** | The set of all valid input instances. |
| **Output** | The required result for every valid input. |

### Finiteness

> [!info]
> Every algorithm must eventually terminate.
>
> It cannot continue executing forever.

### Sequence

> [!info]
> Every instruction must occur in the correct order.
>
> Changing the sequence may produce incorrect results or prevent the algorithm from completing.

### Unambiguity

> [!info]
> Every instruction must be clear and precise.
>
> There should only be **one possible interpretation** of each step.

A well-defined algorithm should specify:

- Where execution begins
- Which data is being processed
- What operation should be performed
- When execution should stop

### Feasibility

> [!info]
> Every operation performed by the algorithm must be achievable using the available resources.

Examples include:

- Available memory
- Processing time
- Hardware limitations
- Input size
