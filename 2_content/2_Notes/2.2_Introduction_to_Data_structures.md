## 2.2.1 What are Data Structures?

> [!info] Definition
> A **data structure** is a way of **organising and storing data** so that it can be managed, accessed and modified efficiently.

Data structures improve the efficiency of common operations such as:

- Searching
- Sorting
- Inserting
- Updating
- Deleting

> [!example]
> Facebook can search through billions of user accounts because user information is organised using efficient data structures.

## Data Structures vs Algorithms

Although often mentioned together, **data structures** and **algorithms** are different concepts.

| Data Structures | Algorithms |
| --------------- | ---------- |
| Organise and store data. | Solve problems through a sequence of steps. |
| Focus on **how data is represented**. | Focus on **how data is processed**. |
| Provide the foundation for algorithms. | Operate on data stored within data structures. |

> [!tip]
> A well-designed algorithm depends on choosing an appropriate data structure.

## Choosing the Right Data Structure

> [!quote]
> "If all you have is a hammer, everything looks like a nail."

Choosing an inappropriate data structure can make solving a problem unnecessarily difficult.

When designing software, always select the structure that best represents the problem.

Examples include:

- Traffic management systems
- University timetabling
- Corporate databases
- Social networks

Most algorithms are designed to operate on a small number of abstract data structures.

## Common Data Structures

### Strings

> [!info]
> A **string** is a sequence of characters used to represent text or patterns.

### Sets

> [!info]
> A **set** is a collection of **unique** items.

```text
{1, 3, 4}

{4, 3, 1}

These represent the same set.
```

### Common Set Operations

| Operation | Description |
| --------- | ----------- |
| Union | Combines two sets |
| Intersection | Finds common elements |
| Difference | Finds elements that exist in one set but not another |

### Trees

> [!info]
> A **tree** represents hierarchical relationships between items.

Trees are commonly used for:

- File systems
- Organisation charts
- Website navigation
- Comments and replies

### Tree Terminology

| Term | Description |
| ---- | ----------- |
| Root | The top node |
| Parent | A node connected above another node |
| Child | A node connected below another node |

## Graphs

> [!info]
> A **graph** represents relationships between connected objects.

Graphs are used whenever a problem involves:

- Networks
- Relationships
- Connections
- Webs

Graphs consist of:

| Component | Description |
| --------- | ----------- |
| Node | An object (vertex) |
| Edge | A connection between two nodes |
